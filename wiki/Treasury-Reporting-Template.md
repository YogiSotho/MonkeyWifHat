# Treasury Reporting Template

## Purpose

This page exists so treasury reporting stays boring, repeatable, and hard to manipulate.

MonkeyWifHat should report treasury activity using the same structure every time.

## Reporting Cadence

Recommended minimum cadence:

- one annual budget framework
- one quarterly treasury update
- one additional public note for any material treasury change

A material change includes any event that would reasonably alter public expectations around runway, wallet balances, or program scope.

## Treasury Sale Disclosure Template

If treasury-controlled tokens are sold or converted, publish a short note covering:

- date of movement:
- wallet used:
- amount moved:
- reason for the sale or conversion:
- destination asset, if relevant:
- effect on runway or budget plan:

This section should not be used to describe initial launch liquidity seeding. That event belongs in the liquidity seeding disclosure, not in treasury sale reporting.

## Reporting Standard

Each report should be understandable by a normal reader.

Do not hide meaning behind dashboards, vague labels, or selective screenshots.

If a sentence would need a follow-up thread to sound honest, rewrite it.
