# Canonical Game Reputation Model

## Purpose

This page explains how MonkeyWifHat could use the token inside a game environment while keeping reputation and canon status separate from raw token ownership.

The design goal is simple:

- the token is the portable economic asset
- reputation is the record of what a player actually did

That is closer to karma than to currency.

## Core Rule

Owning tokens should not automatically make a player important inside the canonical series.

The token can unlock participation, entry, upgrades, themed actions, or optional choices.

Reputation should be what determines who has actually built standing in the world.

## System Hierarchy

The clean order is not reversed. It should stack like this:

1. canonical series
2. game mode or event format inside the series
3. token loop and reputation loop inside that mode

In plain terms:

- the canonical series is the outer world and long-term identity layer
- Hat Arena, Spotlight Board, or Monkey Trials are formats inside that world
- entry, rewards, sinks, and reputation scoring happen inside those formats

So yes: the smaller mechanics sit inside the mode, and the mode sits inside the canonical series.

That is the cleanest model because canon stays larger than any one mini-game.

## Token Role Inside The Game

If the game environment exists, the token can be used for:

- event entry
- optional upgrades or modifiers
- cosmetic claims
- limited canon-linked submissions
- access to premium challenges or themed experiences

The token should enter the environment because the player chooses to use it.

It should not automatically convert into permanent social rank.

## Reputation Role Inside The Game

Reputation should behave more like a visible karma system.

It can track:

- challenge completion
- event placements
- streaks or repeated participation
- recognized creative contributions
- canon appearances or official acknowledgments

Reputation should be:

- earned through activity
- visible under clear rules
- hard to fake
- non-transferable between players

## Why Separate Them

If token balance and reputation are the same thing, the system quickly becomes pay-to-win or pay-to-matter.

That is a weak model for a canonical series.

The stronger model is:

- tokens let players participate economically
- reputation shows what players actually contributed
- canon standing comes from the second, not the first

## Example Reputation Inputs

Good sources of reputation:

- winning official events
- placing in recurring competitions
- maintaining streaks across seasons
- producing selected art, lore, or gameplay contributions
- helping shape recurring series formats under published rules

Weak sources of reputation:

- simply buying more tokens
- making ordinary transfers
- one-time spending with no contribution attached
- opaque moderator favoritism

## Reputation Outputs

If the system is built later, reputation could influence:

- access to advanced event tiers
- eligibility for canon-linked roles
- visibility on leaderboards or spotlight systems
- priority for special submissions
- seasonal recognition inside the series

It should not be marketed as financial yield.

## Faucet And Sink Relationship

The token loop and the reputation loop should interact, but not merge.

Good interaction:

- a player spends tokens to enter a challenge
- the challenge rewards tokens to strong participants
- the player also gains reputation if the participation was meaningful

Bad interaction:

- a player spends tokens and automatically receives social rank
- a player buys status directly through wallet size alone

## Canonical Series Standard

The canonical series should reward history, consistency, and contribution.

That means the project should be able to explain, in plain language:

- how someone enters the series
- how someone earns token rewards
- how someone builds reputation
- how someone becomes canonically important

If those steps are blurry, the system will read as arbitrary.

## Public Explanation

MonkeyWifHat could later let people use the token inside official game experiences, but reputation inside the canonical series should work more like karma than money. Tokens can open doors, fund entries, or support optional actions. Reputation should come from what people actually do, contribute, and achieve over time.
